Security Model

KeyFolio.Client is designed to help humans work with encrypted secrets safely.

Core guarantees

  • No plaintext secrets are written to disk
  • No passphrases are persisted
  • No clipboard access without explicit user action
  • No background logging of sensitive data

Trust boundary

KeyFolio.Client assumes the local machine is trusted during execution. Secrets should still be handled with care.

Reminder: If the machine is compromised, no client app can fully protect secrets typed into it.

Passphrases

  1. Read KEYFOLIO_SECRET from environment
  2. If missing, prompt user
  3. Cache in memory for this session only

Clipboard behavior

Clipboard operations happen only when the user clicks Copy Output. Clipboard contents may be visible to other apps.

Envelope strings

Safe to store/transport, but useless without the correct passphrase.

Common failure modes

  • Passphrase mismatch
  • Different options/version
  • Modified or truncated text

No recovery

Envelopes cannot be decrypted without the original passphrase.
