MailFolio.Console Guide
Common run patterns.

Runbook

Dry run, HTML, attachments, and running the built exe.

Dry run (validate config without sending)

dotnet run -c Release -- `
  --to "someone@agency.mil" `
  --subject "MailFolio Dry Run" `
  --body "This will not be sent." `
  --dryRun

HTML email from file

dotnet run -c Release -- `
  --to "someone@agency.mil" `
  --subject "HTML Test" `
  --bodyFile ".\email.html"

HTML + attachment

dotnet run -c Release -- `
  --to "someone@agency.mil" `
  --subject "Attachment Test" `
  --bodyFile ".\email.html" `
  --attach ".\docs\report.pdf"

Run compiled EXE directly

cd .\bin\Release\net8.0

.\MailFolio.Console.exe `
  --to "someone@agency.mil" `
  --subject "Direct EXE Test" `
  --body "Hello from the compiled MailFolio console."
Tip: When running as a scheduled job, prefer the compiled EXE and set env vars at the machine level.
