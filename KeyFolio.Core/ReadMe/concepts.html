KeyFolio.Core Guide
Core concepts used across implementations.

Concepts

Secret providers, envelope strings, and why KeyFolio.Core is structured this way.

ISecretProvider

ISecretProvider is the abstraction that supplies the secret (typically a passphrase). KeyFolio.Core intentionally does not dictate where secrets come from.

  • Environment variables (MailFolio’s current approach)
  • User prompt / secure input
  • OS-provided secure stores
  • Enterprise vaults (future-ready)

Envelope strings (encrypted text)

An envelope string is an encrypted representation of a secret. Implementations store envelope strings (safe-ish to store) and decrypt them at runtime when needed.

In MailFolio: the key file stores UsernameEnc and PasswordEnc as envelope strings.

KeyFolioOptions

KeyFolioOptions defines crypto parameters (Aad, PBKDF2 iterations, sizes, etc.). It does not hold the passphrase. The passphrase comes from ISecretProvider.

End-to-end flow (how implementations use Core)

Implementation (MailFolio.Console)
  1) Load envelope strings from JSON (UsernameEnc, PasswordEnc)
  2) Create ISecretProvider (EnvSecretProvider uses MAILFOLIO_PASSPHRASE)
  3) Create KeyFolioCrypto(new KeyFolioOptions())
  4) Decrypt(envelopeText, secretProvider) → plaintext
  5) Use plaintext briefly, then discard
Design intent: Apps can swap secret sources without changing crypto logic. KeyFolio.Core stays stable while implementations adapt to their environment.
