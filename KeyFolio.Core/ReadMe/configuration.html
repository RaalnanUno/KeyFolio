MailFolio.Console Guide
Env vars + key file schema + SQLite path.

Configuration

Environment variables, key file JSON, and where SQLite writes.

Environment Variables

NameRequiredDescription
MAILFOLIO_PASSPHRASE Yes Passphrase used by KeyFolio secret provider to decrypt envelope strings.
MAILFOLIO_KEYFILE Yes Absolute path to the JSON key file containing server settings and encrypted creds.
MAILFOLIO_DB No SQLite DB path for failure tracking. If not set, defaults under the executable folder.

Key file JSON schema (example)

{
  "Server": "smtp.example.mil",
  "Port": 587,
  "TlsMode": "StartTls",
  "FromEmail": "no-reply@example.mil",
  "FromName": "MailFolio",
  "UsernameEnc": "<KeyFolio envelope text>",
  "PasswordEnc": "<KeyFolio envelope text>"
}
TLS modes:
  • StartTls (recommended)
  • SslOnConnect
  • None

SQLite “error tracking” database

The DB is written only on failure (connect/auth/send/decrypt/etc.). Successful sends do not create rows.

DB location

  • If MAILFOLIO_DB is set → use that exact file path.
  • Else → bin\Release\net8.0\Data\MailFolio.db (relative to the built output).

Table behavior

  • Table: SentMail
  • Primary Key: ErrorCode
  • Failures upsert by ErrorCode and increment OccurrenceCount.
  • Stored payload is sanitized (no body/subject/username/password).
