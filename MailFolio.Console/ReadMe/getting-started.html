MailFolio.Console Guide
Build and first run (PowerShell).

Getting Started

Start-to-finish: build → verify env → run a test email.

Note: In PowerShell, use # for comments (not @REM).

Build (Release)

dotnet clean
dotnet restore
dotnet build -c Release

Verify environment variables (right where issues usually show up)

Your run attempts “start to finish”. If anything fails later (decrypt/auth), this checkpoint confirms the current process can actually see the values. 

# Confirm the process sees the env vars

[Environment]::GetEnvironmentVariable("MAILFOLIO_PASSPHRASE","Process")
[Environment]::GetEnvironmentVariable("MAILFOLIO_PASSPHRASE","User")
[Environment]::GetEnvironmentVariable("MAILFOLIO_PASSPHRASE","Machine")

[Environment]::GetEnvironmentVariable("MAILFOLIO_KEYFILE","Process")
[Environment]::GetEnvironmentVariable("MAILFOLIO_KEYFILE","User")
[Environment]::GetEnvironmentVariable("MAILFOLIO_KEYFILE","Machine")

Test-Path $env:MAILFOLIO_KEYFILE

Run a plain text test email

dotnet run -c Release -- `
  --to "someone@agency.mil" `
  --subject "MailFolio Test" `
  --body "Hello from MailFolio."
